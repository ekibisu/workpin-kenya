1. Updated Tech Stack
The WorkPin build plan has been fully refactored for Lovable + Supabase as the primary build environment, VS Code with Google Anti Gravity for custom code and integrations, and M-Pesa (Safaricom Daraja API) as the sole payment system — the right choice for the Kenyan market where M-Pesa is used by over 85% of the population.

Why Lovable + Supabase?
Lovable generates a full React + Supabase app from plain-English prompts — cutting initial build time by 60–80%. The output is clean React (Vite + TypeScript + Tailwind) that syncs directly to VS Code via GitHub for fine-tuning. Supabase replaces the entire custom backend: auth, PostgreSQL database, real-time subscriptions (replacing Socket.io), file storage, and serverless Edge Functions — all on one platform with a generous free tier, ideal for an MVP.

Why M-Pesa over Stripe?
Over 85% of Kenyans use M-Pesa as their primary payment method. Card penetration is low and Stripe is not natively supported in Kenya. M-Pesa's Daraja API provides: STK Push (client gets a phone prompt to enter PIN), C2B (customer-to-business paybill), and B2C (business-to-customer for provider payouts). No card form needed anywhere in the app — just a phone number.

Supabase Realtime replaces Socket.io

VS CODE PROMPT 2 — M-Pesa STK Push Edge Function

You are building WorkPin, a Kenyan service marketplace.
Backend: Supabase Edge Functions (Deno + TypeScript).
Payment: Safaricom M-Pesa Daraja API.

Create: supabase/functions/mpesa-stk-push/index.ts

Called when a Provider requests payment or Client books a Quick Pin.

Input body: { workThreadId: string, amountKes: number, phoneNumber: string, description: string }

Steps:
1. Normalize phone: 07XXXXXXXX or +2547XXXXXXXX → 2547XXXXXXXX.
   Validate it starts with 2547 or 2541 (Kenyan numbers only).

2. Get OAuth token:
   GET https://{MPESA_HOST}/oauth/v1/generate?grant_type=client_credentials
   Authorization: Basic base64(MPESA_CONSUMER_KEY:MPESA_CONSUMER_SECRET)
   MPESA_HOST = sandbox.safaricom.co.ke (if MPESA_ENV=sandbox)
              = api.safaricom.co.ke (if MPESA_ENV=production)

3. Generate STK Push password:
   timestamp = new Date().toISOString().replace(/[^0-9]/g,'').slice(0,14)
   password = btoa(MPESA_SHORTCODE + MPESA_PASSKEY + timestamp)

4. POST to /mpesa/stkpush/v1/processrequest:
   {
     BusinessShortCode: MPESA_SHORTCODE,
     Password: password,
     Timestamp: timestamp,
     TransactionType: "CustomerPayBillOnline",
     Amount: Math.ceil(amountKes),
     PartyA: phoneNumber,
     PartyB: MPESA_SHORTCODE,
     PhoneNumber: phoneNumber,
     CallBackURL: Deno.env.get("MPESA_CALLBACK_URL") + "/functions/v1/mpesa-callback",
     AccountReference: "WP-" + workThreadId.slice(0,8).toUpperCase(),
     TransactionDesc: description.slice(0,100)
   }

5. INSERT into payments: { work_thread_id, amount_kes, platform_fee_kes: ceil(amountKes*0.08),
   net_kes: ceil(amountKes*0.92), mpesa_checkout_request_id, status: 'pending' }

6. INSERT system message into messages:
   "Payment request of KES {amount} sent. Check your phone and enter your M-Pesa PIN."

7. Return: { success: true, checkoutRequestId, message: "Check your phone..." }

Error handling: invalid phone → 400, M-Pesa API error → 502 with Daraja error message.
Full TypeScript interfaces for DarajaTokenResponse, DarajaStkPushRequest, DarajaStkPushResponse.
Export CORS headers for Lovable frontend requests.

VS CODE PROMPT 3 — M-Pesa Callback & B2C Payout

You are building WorkPin, a Kenyan service marketplace.
Backend: Supabase Edge Functions (Deno + TypeScript).

FILE 1: supabase/functions/mpesa-callback/index.ts
Receives Safaricom's STK Push result webhook.

Daraja callback structure:
{ Body: { stkCallback: { MerchantRequestID, CheckoutRequestID, ResultCode,
  ResultDesc, CallbackMetadata: { Item: [{Name, Value}] } } } }
ResultCode 0 = success. Anything else = failure.

Logic:
1. Extract CheckoutRequestID from payload.
2. Query payments table for this mpesa_checkout_request_id.
3. If ResultCode === 0:

VS CODE PROMPT 4 — Provider Matching Algorithm

You are building WorkPin, a Kenyan service marketplace.
Backend: Supabase Edge Function (Deno + TypeScript + PostGIS).

Create: supabase/functions/match-providers/index.ts
Called immediately after a client submits a job_request. Returns top 8 matched providers.

Input: { jobRequestId: string }

Step 1: Fetch job_request row (category, lat, lng, budget_min_kes, budget_max_kes, client_id, county).

Step 2: PostGIS geo-query to find candidate providers:
  SELECT pp.*, p.full_name, p.avatar_url, p.phone,
    ST_Distance(
      ST_SetSRID(ST_MakePoint(pp.lng, pp.lat), 4326)::geography,
      ST_SetSRID(ST_MakePoint($jobLng, $jobLat), 4326)::geography
    ) AS distance_m
  FROM provider_profiles pp
  JOIN profiles p ON p.id = pp.user_id
  WHERE $category = ANY(pp.categories)
    AND p.role = 'provider'
    AND ST_DWithin(
      ST_SetSRID(ST_MakePoint(pp.lng, pp.lat), 4326)::geography,
      ST_SetSRID(ST_MakePoint($jobLng, $jobLat), 4326)::geography,
      pp.service_radius_km * 1000
    )
  ORDER BY distance_m ASC LIMIT 20;

Fallback if PostGIS unavailable: Haversine formula in TypeScript.

Step 3: Score each provider (0–100):
  ratingScore   = (pp.avg_rating / 5) * 40      // max 40pts
  responseScore = Math.max(0, 1 - pp.response_time_mins/120) * 30  // max 30pts
  distanceScore = Math.max(0, 1 - distance_m / (pp.service_radius_km*1000)) * 20  // max 20pts
  verifiedBonus = pp.is_verified ? 10 : 0       // max 10pts
  total = ratingScore + responseScore + distanceScore + verifiedBonus

Step 4: Sort by total DESC, take top 8.

Step 5: For each matched provider:
  a. INSERT work_threads: { job_request_id, client_id, provider_id, status:'new' }
  b. INSERT messages (type='system'): "New job near you: {category} in {neighborhood}.
     Client budget: KES {min}–{max}. Tap to send a quote."
  c. INSERT notifications: { user_id: provider_id, type:'new_lead',
     title:'New job request!', body: category + ' in ' + neighborhood }

Step 6: UPDATE job_requests SET status='matched' WHERE id=jobRequestId.

Step 7: Trigger SMS for each provider (call send-sms Edge Function internally).

Return: { matched: N, workThreadIds: [...] }

Edge case: 0 providers found → return { matched: 0 } and set job status to 'open' (not matched).
TypeScript interfaces for all database rows. Use Supabase admin client (service_role key).
